Synopsis: AI Academic Study Copilot & Coach (Updated Final Version)

1. INTRODUCTION

Purpose

This document outlines the complete software requirements and final implementation details for the AI Academic Study Copilot. The application is a full-stack, intelligent study assistant designed to manage learning schedules, quantify study effort, generate interactive materials, and provide context-aware AI tutoring.

Scope

The system provides a centralized dashboard for tracking quantitative study metrics, a dedicated split-pane learning environment with an integrated Pomodoro timer, an AI chat interface grounded in user-specific notes, a syllabus parsing engine, and an advanced gamified activity tracker. The scope encompasses the frontend Single Page Application (SPA), the backend API architecture, the relational database structure, and the AI/RAG (Retrieval-Augmented Generation) pipeline necessary to power the dynamic features.

2. OVERALL DESCRIPTION

Product Perspective

The platform operates on a modern decoupled architecture. The frontend is a responsive React SPA utilizing advanced UI paradigms (custom layout panels, interactive SVG heatmaps, and Recharts). The backend operates via a high-performance Python/FastAPI server, managing state persistence in a PostgreSQL database, while a local embedded vector database (ChromaDB) manages the semantic retrieval required for the AI coach's context awareness.

Target Audience

  • Students: Managing structured university courses, processing dense syllabi, and preparing for examinations through AI-generated quizzes.
  • Self-Learners: Tracking independent study hours, maintaining learning streaks, earning achievement badges, and utilizing on-demand AI tutoring for complex subjects.

3. FUNCTIONAL REQUIREMENTS

Authentication & User Management

  • System Access: Users securely register and log in using email/password (secured via Bcrypt hashing algorithms) or OAuth providers (Google).
  • Profile State: The PostgreSQL backend maintains user profiles, linking them securely to their uploaded syllabi, generated notes, quiz scores, and historical study data via stateless JWT session tokens stored in browser sessionStorage.

Dashboard & Quantitative Analytics

  • Quick Start Protocol: The dashboard features a primary call to action that fetches the user's last active session from the backend and instantly loads the corresponding module.
  • Real-Time Metrics: Displays "Study Time Today," "Current Streak," and "Topics Covered."
  • Study Hours Progress Chart: A graphical Recharts bar chart tracking exact hours studied.
  • Backend Logic: The system records time exclusively based on quantifiable effort (completed Pomodoro intervals or logged active hours). Data is aggregated synchronously via complex PostgreSQL func.sum() queries.

Dynamic Study Session Environment

  • Split Layout: The frontend divides the workspace between study materials (left) and the AI coach (right).
  • Integrated Pomodoro Timer: State management for Work (25 min), Short Break (5 min), and Long Break (15 min) modes. Syncing requires completed work intervals to send a payload to the backend to update daily study hours.
  • Left Panel (Study Materials): Interactive multiple-choice Quizzes provide dynamic assessments rendered from structured JSON payloads.
  • Right Panel (AI Study Coach): A persistent, real-time chat interface. The UI utilizes react-markdown and remark-gfm Abstract Syntax Tree (AST) parsers to render streaming Server-Sent Events (SSE) from the AI in real-time, displaying complex math equations and code blocks smoothly.
  • Context Awareness: Every message sent to the AI bundles metadata about the active topic. The backend RAG pipeline injects this specific context into the Gemini prompt to ensure highly relevant, localized tutoring without hallucination.

Syllabus Management & Ingestion Pipeline

  • Upload Pipeline: The UI accepts PDF files via a dedicated dropzone.
  • Processing Engine: The backend extracts raw text from the uploaded document. A LangChain orchestration pipeline utilizes Google Gemini 2.5 Flash to structure the raw text into a hierarchical JSON format (Units, Topics). The structured outline is saved to the PostgreSQL database and populates the user's course navigation menu.

Contribution & Gamification Engine

  • Activity Heat Map: A 365-day SVG contribution matrix (similar to GitHub) visualizing daily study intensity. The backend API returns an array of date-value pairs representing study duration.
  • Streak & Badge Calculation: The database evaluates consecutive days where Study_Hours > 0 to update the user's "Current Streak." Additionally, completing topics and maintaining streaks unlocks specific Achievement Badges within the PostgreSQL database to further incentivize learning.

4. NON-FUNCTIONAL REQUIREMENTS

UI/UX & Theming

  • Theme Support: Strict UI components driven by Tailwind CSS styling constraints.
  • Responsive Degradation: The layout collapses gracefully into a stacked single-column view on mobile devices, and non-essential UI elements are hidden on narrow viewports.

System Performance & Scalability

  • Timer Precision: The Pomodoro timer relies on absolute system clock timestamps (Date.now()) rather than pure interval ticks to prevent browser-throttling drift.
  • API Latency: Standard CRUD operations and analytics retrieval resolve in under 200ms.
  • Streaming Responses: The AI Coach chat interface utilizes Server-Sent Events (SSE) via Python yield generators to stream text responses chunk-by-chunk over HTTP, ensuring immediate visual feedback.

Security & Data Integrity

  • Payload Validation: All incoming API requests are strictly validated against predefined Pydantic schemas to prevent injection attacks and server crashes.
  • Data Isolation: Database queries enforce tenant isolation via JWT payload extraction, ensuring users can only access their own uploaded syllabi and notes.

5. TECHNOLOGY STACK

Frontend Architecture

  • Core Framework: React 18+
  • Routing: React Router
  • Styling: Tailwind CSS
  • Icons: Lucide React
  • Complex UI State: Global React Context API linked to Browser sessionStorage

Backend & Data Architecture

  • Core Framework: Python with FastAPI (for high-performance, asynchronous endpoints via Uvicorn).
  • Primary Database: PostgreSQL (managed via SQLAlchemy ORM for relational data, users, badges, and quizzes).
  • Vector Database: ChromaDB (embedded local SQLite implementation for storing and querying text embeddings of uploaded syllabi).
  • AI Orchestration: Pure LangChain (utilizing GoogleGenerativeAI models gemini-2.5-flash and text-embedding-004). Note: LangGraph was explicitly excluded to reduce orchestration overhead and meet API latency requirements.
